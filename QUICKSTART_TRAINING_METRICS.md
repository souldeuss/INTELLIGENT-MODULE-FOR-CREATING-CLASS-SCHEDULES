# 🚀 Швидкий старт: Метрики навчання

## Як переглянути метрики навчання нейромережі

### Спосіб 1: Через AI Generator (Рекомендовано)

1. Перейдіть на сторінку **AI Генератор** (ліве меню → AI Генератор)
2. Запустіть генерацію розкладу натиснувши кнопку **"Запустити генерацію"**
3. Дочекайтеся завершення навчання (progress bar покаже 100%)
4. Натисніть кнопку **"Метрики навчання"** внизу панелі керування
5. Перегляньте 4 графіки з детальною статистикою

### Спосіб 2: Через меню навігації

1. У лівому меню розгорніть секцію **"Аналітика"**
2. Натисніть на **"Метрики навчання"**
3. Переглядайте графіки останнього навчання

### Спосіб 3: Прямий URL

Відкрийте у браузері: `http://localhost:3000/training-metrics`

## Що показують графіки?

### 📊 Інформаційна панель (верх екрану)

- **Дата навчання** - коли модель навчалась востаннє
- **Кількість епох** - скільки епох було виконано
- **Фінальна винагорода** - остання винагорода (чим більше, тим краще)
- **Завершеність розкладу** - відсоток заповнених слотів (має бути близько 100%)

### 🎯 Графік 1: Винагороди за епохами

**Що це?** Показує, наскільки "хороший" розклад генерує модель на кожній епосі.

✅ **Хороший результат:** Стовпці зростають з епохами  
⚠️ **Проблема:** Винагорода падає або не змінюється

### ⚠️ Графік 2: Порушення обмежень за епохами

**Що це?** Кількість конфліктів у розкладі.

- **Червона лінія** - жорсткі порушення (конфлікти викладачів/груп)
- **Помаранчева лінія** - м'які порушення (невідповідність типу аудиторії тощо)

✅ **Хороший результат:** Обидві лінії йдуть вниз  
⚠️ **Проблема:** Порушення залишаються високими

### ✅ Графік 3: Завершеність розкладу за епохами

**Що це?** Відсоток заповнених класів у розкладі.

✅ **Хороший результат:** Лінія наближається до 100%  
⚠️ **Проблема:** Застрягає нижче 80-90%

### 📉 Графік 4: Втрати моделі за епохами

**Що це?** Показує, наскільки добре навчається нейромережа.

- **Фіолетова лінія** - втрати Actor (політика дій)
- **Синя лінія** - втрати Critic (оцінка стану)

✅ **Хороший результат:** Лінії зменшуються або стабілізуються  
⚠️ **Проблема:** Втрати зростають (можливо overfitting)

## Коли переглядати метрики?

### ✅ Перегляд метрик після генерації

Після кожної генерації розкладу:

1. Перевірте, чи завершеність > 95%
2. Подивіться, чи порушення близькі до 0
3. Оцініть стабільність навчання

### 🔧 Налаштування гіперпараметрів

Якщо результати погані:

1. Зменште **Learning Rate** (якщо втрати зростають)
2. Збільште **Batch Size** (якщо навчання нестабільне)
3. Збільште **Кількість епох** (якщо модель не встигла навчитись)
4. Перегляньте **Ваги обмежень** (якщо багато порушень)

### 📈 Порівняння різних запусків

Зберігайте скріншоти метрик щоб порівняти:

- Різні налаштування гіперпараметрів
- Різні розміри даних
- Вплив фіксованих класів (Preserve Locked)

## Поради щодо інтерпретації

### Сценарій 1: Ідеальне навчання ✅

```
Епохи: 100
Винагорода: 20 → 90 (плавне зростання)
Жорсткі порушення: 15 → 0
М'які порушення: 50 → 5
Завершеність: 70% → 99%
Втрати: стабільне зменшення
```

**Дія:** Використовуйте згенерований розклад!

### Сценарій 2: Повільне навчання 🐌

```
Епохи: 100
Винагорода: 10 → 35 (повільно зростає)
Порушення: зменшуються, але повільно
Завершеність: 85% → 92%
```

**Дія:** Збільште кількість епох до 200-300

### Сценарій 3: Нестабільне навчання ⚠️

```
Винагорода: +30 → -10 → +50 (коливається)
Порушення: 10 → 25 → 8 (хаотично)
Втрати: різко зростають після епохи 50
```

**Дія:**

- Зменште Learning Rate (0.0003 → 0.0001)
- Збільште Batch Size (64 → 128)
- Додайте більше даних

### Сценарій 4: Overfitting 📈📉

```
Епохи: 1-80 → добре навчання
Епохи: 81-150 → результати погіршуються
Втрати: спочатку ↓, потім ↑
```

**Дія:**

- Зупиніть навчання раніше (використайте checkpoint епохи ~80)
- Збільште регуляризацію
- Додайте більше різноманітних даних

## FAQ

### ❓ Метрики не відображаються

**Причина:** Модель ще не навчалась після запуску системи.  
**Рішення:** Згенеруйте хоча б один розклад через AI Generator.

### ❓ Графіки показують погані результати

**Можливі причини:**

1. Дані занадто складні (багато курсів, мало аудиторій)
2. Недостатня кількість епох навчання
3. Невідповідні гіперпараметри

**Рішення:** Див. секцію "Налаштування гіперпараметрів" вище.

### ❓ Як зберегти графіки?

**Рішення:** Зробіть скріншот екрану (Win+Shift+S) або натисніть PrintScreen.

### ❓ Метрики оновлюються в реальному часі?

**Поточна версія:** Ні, показує результати останнього завершеного навчання.  
**Майбутня версія:** Планується WebSocket для live-оновлень.
